## 规则类型概览

系统支持两种主要规则类型:

| 类型 | 定义位置 | 用途 |
|------|----------|------|
| AlertRule | `internal/rules/rule.go:21` | 单事件匹配告警 |
| CorrelationRule | `internal/rules/rule.go:66` | 多事件关联分析 |

## AlertRule 告警规则

AlertRule 是告警引擎的核心配置,定义在 `internal/rules/rule.go:21-43`:

```go
type AlertRule struct {
    Name           string         `yaml:"name"`
    Description    string         `yaml:"description"`
    Enabled        bool           `yaml:"enabled"`
    Severity       types.Severity `yaml:"severity"`
    Score          float64        `yaml:"score"`
    MitreAttack    string         `yaml:"mitre_attack,omitempty"`
    Priority       int            `yaml:"priority"`    // 1-100,默认 50
    Weight         float64        `yaml:"weight"`      // 告警权重,默认 1.0
    Filter         *Filter        `yaml:"filter"`
    Conditions     *Conditions    `yaml:"conditions,omitempty"`
    Threshold      int            `yaml:"threshold,omitempty"`
    TimeWindow     time.Duration  `yaml:"time_window,omitempty"`
    AggregationKey string         `yaml:"aggregation_key,omitempty"`
    Message        string         `yaml:"message"`
    Tags           []string       `yaml:"tags,omitempty"`
    Status         string         `yaml:"status,omitempty"`
    Product        string         `yaml:"logsource,omitempty"`
    LogSource      *LogSource     `yaml:"logsource,omitempty"`
    FalsePositives []string       `yaml:"falsepositives,omitempty"`
    Level          string         `yaml:"level,omitempty"`
    References     []string       `yaml:"references,omitempty"`
}
```

### 核心字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `Name` | string | 规则唯一标识符 |
| `Severity` | types.Severity | 严重级别 (critical/high/medium/low/info) |
| `Score` | float64 | 规则基础分数 |
| `MitreAttack` | string | MITRE ATT&CK 技术 ID (如 T1110) |
| `Priority` | int | 执行优先级 (1-100),越高越先执行 |
| `Weight` | float64 | 告警权重,用于计算有效分数 |
| `Filter` | *Filter | 事件过滤器 |
| `Conditions` | *Conditions | 高级条件组 |
| `Threshold` | int | 触发阈值 (需要匹配的次数) |
| `TimeWindow` | time.Duration | 阈值时间窗口 |
| `AggregationKey` | string | 聚合键 (user/computer/source/ip) |
| `Message` | string | 告警消息模板,支持变量替换 |

### 消息模板变量

规则支持在 Message 中使用模板变量 (`internal/rules/rule.go:45-64`):

```go
func (r *AlertRule) BuildMessage(event *types.Event) string {
    msg := r.Message
    msg = strings.ReplaceAll(msg, "{{.EventID}}", fmt.Sprintf("%d", event.EventID))
    msg = strings.ReplaceAll(msg, "{{.Source}}", event.Source)
    msg = strings.ReplaceAll(msg, "{{.Computer}}", event.Computer)
    msg = strings.ReplaceAll(msg, "{{.User}}", userStr)
    msg = strings.ReplaceAll(msg, "{{.Message}}", event.Message)
    return msg
}
```

可用变量: `{{.EventID}}`, `{{.Source}}`, `{{.Computer}}`, `{{.User}}`, `{{.Message}}`

### 有效分数计算

```go
func (r *AlertRule) GetEffectiveScore() float64 {
    return r.Score * r.GetWeight()
}
```

## Filter 过滤器

Filter 用于对事件进行基础过滤,定义在 `internal/rules/rule.go:89-105`:

```go
type Filter struct {
    EventIDs         []int32          `yaml:"event_ids,omitempty"`
    Levels           []string         `yaml:"levels,omitempty"`
    LogNames         []string         `yaml:"log_names,omitempty"`
    Sources          []string         `yaml:"sources,omitempty"`
    Computers        []string         `yaml:"computers,omitempty"`
    Keywords         string           `yaml:"keywords,omitempty"`
    KeywordMode      LogicalOp        `yaml:"keyword_mode,omitempty"`
    TimeRange        *types.TimeRange `yaml:"time_range,omitempty"`
    LogonTypes       []int            `yaml:"logon_types,omitempty"`
    ExcludeUsers     []string         `yaml:"exclude_users,omitempty"`
    ExcludeComputers []string         `yaml:"exclude_computers,omitempty"`
    ExcludeDomains   []string         `yaml:"exclude_domains,omitempty"`
    MinFailureCount  int              `yaml:"min_failure_count,omitempty"`
    IpAddress        []string         `yaml:"ip_address,omitempty"`
    ProcessNames     []string         `yaml:"process_names,omitempty"`
}
```

### 关键字匹配模式

```go
type LogicalOp string

const (
    OpAnd LogicalOp = "AND"
    OpOr  LogicalOp = "OR"
)
```

- `AND` 模式: 所有关键字都必须匹配
- `OR` 模式: 任一关键字匹配即可

## CorrelationRule 关联规则

用于检测多事件之间的关联关系,定义在 `internal/rules/rule.go:66-76`:

```go
type CorrelationRule struct {
    Name        string         `yaml:"name"`
    Description string         `yaml:"description"`
    Enabled     bool           `yaml:"enabled"`
    Severity    types.Severity `yaml:"severity"`
    Patterns    []*Pattern     `yaml:"patterns"`
    TimeWindow  time.Duration  `yaml:"time_window"`
    Join        string         `yaml:"join"`
    MitreAttack string         `yaml:"mitre_attack,omitempty"`
    Tags        []string       `yaml:"tags,omitempty"`
}
```

### Join 策略

| 策略 | 说明 |
|------|------|
| `user` | 按用户关联 (User 或 UserSID) |
| `computer` | 按计算机名关联 |
| `ip` | 按 IP 地址关联 |
| 空字符串 | 仅按时间顺序关联 |

## Pattern 模式定义

Pattern 定义关联规则中的单个事件模式 (`internal/rules/rule.go:78-87`):

```go
type Pattern struct {
    EventID    int32         `yaml:"event_id"`
    Conditions []*Condition  `yaml:"conditions,omitempty"`
    Join       string        `yaml:"join,omitempty"`
    TimeWindow time.Duration `yaml:"time_window,omitempty"`
    MinCount   int           `yaml:"min_count,omitempty"`
    MaxCount   int           `yaml:"max_count,omitempty"`
    Ordered    bool          `yaml:"ordered,omitempty"`
    Negate     bool          `yaml:"negate,omitempty"`
}
```

| 字段 | 说明 |
|------|------|
| `EventID` | 目标 Windows 事件 ID |
| `Conditions` | 附加条件 |
| `MinCount` | 最小匹配数 |
| `MaxCount` | 最大匹配数 |
| `Ordered` | 是否要求事件按顺序发生 |
| `Negate` | 是否取反 (该模式不应出现) |

## MITRE ATT&CK 映射

### 规则中的映射

每条规则通过 `MitreAttack` 字段关联到 MITRE ATT&CK 技术 ID (`internal/rules/rule.go:27`):

```go
MitreAttack    string         `yaml:"mitre_attack,omitempty"`
```

### ID 格式验证

系统验证 MITRE ID 格式 (`internal/rules/rule.go:333-360`):

```go
var mitreIDRegex = regexp.MustCompile(`^(T\d{4}(?:\.\d{3})?)$`)

func validateMitreIDFormat(id string) error {
    if !mitreIDRegex.MatchString(id) {
        return fmt.Errorf("invalid mitre_attack format: %s (expected T#### or T####.###)", id)
    }
    // 验证战术类型 (1-3)
    tacticType := tacticNum / 1000
    if tacticType < 1 || tacticType > 3 {
        return fmt.Errorf("invalid mitre_attack: %s (tactic type must be 1-3)", id)
    }
    return nil
}
```

有效格式示例:
- `T1110` - Brute Force
- `T1078.004` - Cloud Accounts
- `T1059.001` - PowerShell

## 规则验证与排序

### 规则验证

AlertRule 在加载前会进行完整验证 (`internal/rules/rule.go:172-217`):

```go
func (r *AlertRule) Validate() error {
    if r.Name == "" {
        return fmt.Errorf("rule name is required")
    }
    if r.Severity == "" {
        return fmt.Errorf("severity is required")
    }
    if r.Filter == nil && r.Conditions == nil {
        return fmt.Errorf("either filter or conditions is required")
    }
    if r.Threshold > 0 && r.TimeWindow == 0 {
        return fmt.Errorf("threshold requires time_window to be set")
    }
    // ... 验证 Filter 和 Conditions
}
```

### 规则排序

规则按 Priority 和 Weight 排序,高优先级的规则先执行 (`internal/rules/rule.go:383-407`):

```go
type RuleSorter struct {
    rules []*AlertRule
}

func (s *RuleSorter) Less(i, j int) bool {
    pi := s.rules[i].GetPriority()
    pj := s.rules[j].GetPriority()
    if pi != pj {
        return pi > pj  // Priority 高的在前
    }
    return s.rules[i].GetWeight() > s.rules[j].GetWeight()  // Weight 高的在前
}
```

## 内置规则示例

内置规则定义在 `internal/rules/builtin/definitions.go` 中,以下是几个典型示例:

### 失败登录阈值检测

```go
{
    Name:           "failed-login-threshold",
    Description:    "失败登录次数超过阈值",
    Enabled:        true,
    Severity:       types.SeverityHigh,
    Score:          70,
    MitreAttack:    "T1110",
    Threshold:      10,
    TimeWindow:     5 * time.Minute,
    AggregationKey: "user",
    Filter: &rules.Filter{
        EventIDs: []int32{4625},
        Levels:   []string{"Error"},
    },
    Message: "Failed login threshold exceeded: {{.Count}} failed attempts",
    Tags:    []string{"authentication", "brute-force"},
}
```

### Mimikatz 检测

```go
{
    Name:        "mimikatz-suspect",
    Description: "Mimikatz 相关工具执行",
    Enabled:     true,
    Severity:    types.SeverityCritical,
    Score:       100,
    MitreAttack: "T1003",
    Filter: &rules.Filter{
        EventIDs: []int32{1},
        Levels:   []string{"Info"},
    },
    Conditions: &rules.Conditions{
        Any: []*rules.Condition{
            {Field: "process_name", Operator: "contains", Value: "mimikatz"},
            {Field: "command_line", Operator: "contains", Value: "sekurlsa"},
            {Field: "command_line", Operator: "contains", Value: "lsadump"},
        },
        None: []*rules.Condition{
            {Field: "process_name", Operator: "contains", Value: "tasklist"},
        },
    },
    Message: "Mimikatz or credential dumping tool detected",
    Tags:    []string{"credential-access", "malware"},
}
```

## 一、模块化架构总览

项目采用标准的 Go 项目布局 (`cmd/internal/pkg`)，模块划分清晰，职责明确：

| 层级 | 模块 | 职责 |
|------|------|------|
| **核心层** | engine, parsers, storage | 日志解析、数据存储、导入引擎 |
| **分析层** | alerts, rules, correlation, ueba | 告警检测、规则匹配、关联分析、UEBA |
| **功能层** | forensics, persistence, reports | 取证、持久化检测、报告生成 |
| **接口层** | api, tui | HTTP API、TUI 界面 |
| **支撑层** | types, config, observability | 类型定义、配置、可观测性 |

## 二、分模块详细评估

### 2.1 EVTX 解析模块 (`pkg/evtx`)

#### 评估结果

| 维度 | 评分 | 说明 |
|------|------|------|
| **代码质量** | ⭐⭐⭐⭐ | 结构清晰，实现完整 |
| **架构设计** | ⭐⭐⭐ | 基础实现可用，但存在改进空间 |
| **功能完整性** | ⭐⭐⭐⭐ | 支持多种解析方式 (FastEvents, XML, Chunk) |
| **性能** | ⭐⭐⭐ | 基本满足需求，可优化 |
| **规范性** | ⭐⭐⭐⭐ | 遵循 Go 惯例 |

#### 优点

1. **多解析策略**: 提供了 `FastEvents()`, `ParseXML()`, `ReadChunk()` 三种解析方式，可适应不同场景
2. **记录扫描器**: `RecordScanner` 实现流式读取，适合大文件处理
3. **类型定义完整**: `Header`, `Chunk`, `Record`, `XMLRecord` 等结构清晰
4. **错误处理规范**: 定义了专用错误类型 (`ErrInvalidMagic`, `ErrInvalidHeader`, `ErrInvalidChunk`)

#### 缺点

1. **XML 解析冗余**: `parseRecord()` 函数内同时进行二进制解析和 XML 解析，逻辑混乱
2. **未使用外部库**: 项目在 `internal/parsers/evtx/parser.go` 中使用了 `github.com/0xrawsec/golang-evtx`，但 `pkg/evtx` 是自研实现，两者功能重叠
3. **时间戳处理问题**: `time.Unix(0, timestamp*100)` 假设纳秒精度，但 Windows EVTX 使用 100 纳秒间隔
4. **二进制解析基础**: 仅解析了固定偏移，未处理复杂的 Binary XML 格式

#### 改进方向

1. 统一使用外部成熟的 EVTX 解析库，移除重复实现
2. 重构 `parseRecord()`，分离二进制解析和 XML 解析逻辑
3. 修正时间戳转换算法
4. 增加对更多 EVTX 变体的支持

| **功能完整性** | ⭐⭐⭐⭐⭐ | 功能完整 |
| **性能** | ⭐⭐⭐⭐ | WAL 模式，批量写入 |
| **规范性** | ⭐⭐⭐⭐ | 遵循 SQLite 最佳实践 |

#### 优点

1. **WAL 模式**: 启用 WAL 提高并发读写性能
2. **批量插入**: `InsertBatch()` 使用事务和预处理语句
3. **Repository 模式**: `EventRepo`, `AlertRepo` 封装清晰
4. **参数化查询**: 有效防止 SQL 注入

#### 缺点

1. **查询构建方式不安全**: `EventRepo.Search()` 中使用了字符串格式化构建 SQL
   ```go
   conditions = append(conditions, fmt.Sprintf("message GLOB '%s'", req.Keywords))
   ```
   这会导致 **SQL 注入漏洞**
2. **连接池配置**: `SetMaxOpenConns(1)` 虽然解决 SQLite 并发限制，但过于保守
3. **重复扫描函数**: `scanEvent()` 和 `scanEventFromRows()` 逻辑完全相同
4. **索引缺失**: 大量字段缺少索引，影响查询性能

#### 改进方向

1. **紧急修复**: 修复 SQL 注入漏洞，改用参数化查询
2. 优化连接池配置，或考虑使用更高性能的数据库
3. 合并重复的扫描函数
4. 增加必要的索引（`event_id`, `timestamp`, `level`, `source`）

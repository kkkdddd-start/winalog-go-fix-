## 一、安全日志补充

### 4673 - 特权服务请求

**级别**: 审核成功  
**描述**: 账户请求了特权服务操作。此事件在用户尝试执行需要特定权限的系统级操作时生成，通常与 4674（特权对象操作）配对出现。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserSid | 请求特权的账户 SID | 确认操作者 |
| SubjectUserName | 账户名 | 非预期账户需告警 |
| SubjectLogonId | 登录会话 ID | 关联 4624 登录事件 |
| PrivilegeList | 请求的特权列表 | 分析请求的权限类型 |
| Service | 请求的服务名称 | 识别被操作的服务 |

**应急场景**:
- **提权检测**: 非管理员账户请求 SeDebugPrivilege → 可能是提权攻击
- **服务篡改**: 请求 SeLoadDriverPrivilege + 服务操作 → 可能是恶意驱动加载
- **权限滥用**: 普通用户频繁请求特权服务 → 账户已被攻陷
- **关联分析**: 4673 → 4674 序列表示特权服务请求后执行了特权操作

### 4674 - 对特权对象操作

**级别**: 审核成功  
**描述**: 对具有特殊权限的对象执行了操作。此事件在用户尝试修改、读取或删除受保护的系统对象时生成，是检测权限滥用和横向移动的重要指标。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| ObjectServer | 对象服务器 (Security/LSA 等) | 识别操作的目标系统 |
| ObjectType | 对象类型 (File/Key/Token 等) | 分析操作意图 |
| ObjectName | 对象名称/路径 | 确认被操作的对象 |
| OperationType | 操作类型 (%%1536=创建, %%1537=读取, %%1538=写入, %%1539=删除, %%1540=执行) | 分析具体操作 |
| AccessMask | 请求的访问权限 (0x1=读, 0x2=写, 0x4=执行, 0x8=删除) | 评估操作严重性 |
| SubjectUserName | 执行操作的账户 | 确认操作者 |

**AccessMask 常见值**:

| 值 | 权限 | 说明 |
|----|------|------|
| 0x00000001 | FILE_READ_DATA | 读取文件数据 |
| 0x00000002 | FILE_WRITE_DATA | 写入文件数据 |
| 0x00000004 | FILE_APPEND_DATA | 追加文件数据 |
| 0x00000020 | FILE_EXECUTE | 执行文件 |
| 0x00010000 | DELETE | 删除对象 |
| 0x00040000 | WRITE_DAC | 修改访问控制 |
| 0x00080000 | WRITE_OWNER | 修改所有者 |

**应急场景**:
- **凭据窃取**: 访问 lsass.exe 进程 (ObjectType=Process, AccessMask=0x10) → 可能是 Mimikatz
- **SAM 数据库访问**: 访问 \Device\HarddiskVolume*\Windows\System32\config\SAM → 可能是密码提取
- **注册表篡改**: 修改 HKLM\SYSTEM\CurrentControlSet\Services → 可能是服务持久化
- **权限变更**: WRITE_DAC 或 WRITE_OWNER 操作 → 攻击者修改对象权限
- **数据外泄**: 大量读取操作指向敏感文件 (NTDS.dit, 密码文件)

### 4689 - 进程退出

**级别**: 审核成功  
**描述**: 进程已退出。此事件与 4688（进程创建）配对使用，用于追踪进程的完整生命周期。注意：此事件需要启用"审核进程终止"审核策略。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserSid | 退出进程的账户 SID | 确认进程所有者 |
| SubjectUserName | 账户名 | 非系统账户退出需关注 |
| SubjectLogonId | 登录会话 ID | 关联用户会话 |
| ProcessId | 进程 PID | 关联 4688 的 NewProcessId |
| NewProcessName | 退出的进程路径 | 识别退出的程序 |
| ExitStatus | 退出状态码 (0=正常) | 非零退出码表示异常 |

**ExitStatus 常见值**:

| 值 | 说明 |
|----|------|
| 0 | 正常退出 (STATUS_SUCCESS) |
| 1 | 通用错误 |
| 0xC0000005 | 访问违规 (Access Violation) |
| 0xC000013A | 被 Ctrl+C 终止 |
| 0xC0000409 | 堆栈缓冲区溢出 |

**应急场景**:
- **异常退出**: 关键系统进程 (如 svchost.exe, lsass.exe) 非零退出 → 可能是崩溃或攻击
- **进程生命周期**: 4688 → ... → 4689 可计算进程运行时长
- **恶意软件检测**: 短生命周期进程 (几秒内 4688→4689) + 异常路径 → 可能是 Dropper
- **进程替换**: 正常进程退出后立即出现同名新进程 → 可能是进程注入后重启

### 4722 - 账户已启用

**级别**: 审核成功  
**描述**: 禁用的用户账户被重新启用。此事件仅在域控制器上生成（域账户）或在本地计算机上生成（本地账户）。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserName | 执行启用操作的管理员账户 | 确认操作者 |
| TargetUserName | 被启用的账户 | 确认目标账户 |
| TargetDomainName | 账户所在域 | 域/本地账户 |

**应急场景**:
- **休眠账户激活**: 长期禁用的管理员账户突然被启用 → 可能是攻击者重新激活休眠后门
- **离职员工账户**: 已离职员工的账户被启用 → 内部威胁或违规操作
- **关联检测**: 4722 → 4624 (短时间内) → 启用后立即登录，可能是攻击者在利用
- **合规审计**: 所有账户启用操作都应记录并审批

### 4723 - 尝试更改账户密码

**级别**: 审核成功/失败  
**描述**: 用户尝试更改自己的密码。此事件记录了密码更改的尝试，无论成功还是失败。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserName | 尝试更改密码的账户 | 确认操作者 |
| TargetUserName | 密码被更改的账户 | 通常是同一账户 |
| Status | 状态码 (0x0=成功) | 分析结果 |

**常见 Status 值**:

| 值 | 说明 |
|----|------|
| 0x0 | 密码更改成功 |
| 0xC000006A | 旧密码不正确 |
| 0xC0000071 | 密码已过期 |
| 0xC000038D | 密码不满足复杂度要求 |

**应急场景**:
- **攻击者密码更改**: 管理员账户密码被非预期用户更改 → 账户已被攻陷
- **密码策略绕过**: 多次 4723 失败 → 可能是攻击者尝试设置弱密码
- **关联检测**: 4624 (登录) → 4723 (改密) → 可能是攻击者在锁定账户访问权

### 4770 - TGS 更新

**级别**: 审核成功

**应急场景**:

- **服务票据续期**: 高权限服务票据续期 → 可能是攻击者保持访问权

| 值 | 说明 |
|----|------|

## 八、补充密码和凭据事件

### 4724 - 尝试重置账户密码 (详细版)

**级别**: 审核成功  
**描述**: 管理员或其他授权账户重置了用户账户的密码。此事件与 4723（用户自己更改密码）不同，4724 记录的是由他人执行的密码重置操作。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserName | 执行重置的管理员 | 确认操作者 |
| SubjectDomainName | 管理员所在域 | 域/本地操作 |
| TargetUserName | 密码被重置的账户 | 确认目标账户 |
| TargetDomainName | 目标账户所在域 | 域/本地账户 |
| Status | 状态码 (0x0=成功) | 分析结果 |

**常见 Status 值**:

| 值 | 说明 |
|----|------|
| 0x0 | 密码重置成功 |
| 0xC0000064 | 目标账户不存在 |
| 0xC0000022 | 权限不足 |

**应急场景**:
- **攻击者夺权**: 非预期管理员重置高权限账户密码 → 账户已被攻陷
- **内部威胁**: IT 员工重置其他管理员账户密码 → 可能是权限提升
- **批量重置**: 短时间内多个账户密码被重置 → 可能是攻击者在批量控制账户
- **关联检测**: 4624 (登录) → 4724 (重置密码) → 攻击者登录后重置密码锁定账户
- **合规审计**: 所有密码重置操作都应记录原因和审批
- **与 4723 区别**: 4723 是用户自己改密，4724 是他人重置 → 4724 风险更高

---

### 5058 - 密钥管理操作

**级别**: 信息  
**描述**: 密钥管理操作。此事件在 CNG (Cryptography Next Generation) 或 CryptoAPI 执行密钥创建、导入、导出等操作时生成。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| ProcessName | 执行操作的进程 | 确认操作来源 |
| Operation | 操作类型 (创建、导入、导出、删除) | 分析密钥操作 |
| KeyType | 密钥类型 | 分析密钥用途 |

**应急场景**:
- **密钥导出**: 导出私钥 → 可能是攻击者窃取加密材料
- **密钥创建**: 非预期进程创建密钥 → 可能是恶意软件
- **密钥删除**: 删除关键密钥 → 可能是攻击者破坏加密系统
- **关联检测**: 5058 → 5061 (加密操作) → 密钥操作后使用

---

### 5059 - 密钥迁移操作

**级别**: 信息  
**描述**: 密钥迁移操作。此事件在密钥从一个存储位置迁移到另一个位置时生成。

**应急场景**:
- **密钥迁移**: 密钥被迁移到非标准存储 → 可能是攻击者窃取
- **合规审计**: 所有密钥迁移操作都应记录

---

### 5061 - 加密操作

**级别**: 信息  
**描述**: 加密操作。此事件在 CNG 执行加密、解密、签名、验证等操作时生成。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| ProcessName | 执行操作的进程 | 确认操作来源 |
| Operation | 操作类型 (加密、解密、签名、验证) | 分析加密操作 |
| KeyName | 使用的密钥名称 | 确认使用的密钥 |

**应急场景**:
- **异常加密**: 非预期进程执行大量加密操作 → 可能是勒索软件
- **签名伪造**: 异常签名操作 → 可能是攻击者伪造证书
- **关联检测**: 5061 → 5145 (文件访问) → 加密后访问文件

## 九、补充计划任务事件

### 4699 - 计划任务删除 (详细版)

**级别**: 审核成功  
**描述**: 计划任务被删除。此事件在计划任务被永久删除时生成，是检测攻击者痕迹清理和持久化机制移除的关键事件。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserName | 删除账户 | 确认操作者 |
| TaskName | 被删除的任务名 | 确认被删除的任务 |
| TaskContent | 任务内容 (删除前) | 分析被删除的任务 |

**应急场景**:
- **痕迹清理**: 攻击者删除恶意计划任务 → 4698 (创建) → 使用 → 4699 (删除)
- **安全软件恢复**: 管理员删除攻击者创建的恶意任务
- **关联检测**: 4699 前有大量 4688 (进程创建) → 可能是任务执行后被删除
- **审计要求**: 所有计划任务删除操作都应记录原因

---

### 4700 - 计划任务启用 (详细版)

**级别**: 审核成功  
**描述**: 计划任务被启用。此事件在禁用的计划任务被重新启用时生成，是检测攻击者激活休眠持久化机制的关键事件。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserName | 启用账户 | 确认操作者 |
| TaskName | 被启用的任务名 | 确认被启用的任务 |
| TaskContent | 任务内容 | 分析任务配置 |

**应急场景**:
- **休眠激活**: 长期禁用的任务突然被启用 → 可能是攻击者激活后门
- **安全软件恢复**: 攻击者禁用安全相关任务后被管理员重新启用
- **关联检测**: 4700 → 4688 (进程创建) → 任务启用后立即执行
- **审计要求**: 所有计划任务启用操作都应记录原因

---

### 4701 - 计划任务禁用 (详细版)

**级别**: 审核成功  
**描述**: 计划任务被禁用。此事件在计划任务被禁用时生成，是检测攻击者关闭安全监控和日志收集的关键事件。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserName | 禁用账户 | 确认操作者 |
| TaskName | 被禁用的任务名 | 确认被禁用的任务 |

**应急场景**:
- **关闭监控**: 攻击者禁用安全扫描、日志收集等任务 → 降低检测能力
- **维护操作**: 正常维护期间禁用任务 → 需要审批记录
- **关联检测**: 4701 前有 4624 (登录) → 可能是攻击者登录后关闭监控
- **审计要求**: 所有计划任务禁用操作都应记录原因

## 十、补充组成员变更事件

### 4756 - 通用组成员添加

**级别**: 审核成功  
**描述**: 成员被添加到通用安全组。此事件在 Active Directory 中通用组成员变更时生成，是检测跨域权限提升的关键事件。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| SubjectUserName | 添加账户 | 确认操作者 |
| TargetUserName | 目标组名 | 确认被添加的组 |
| MemberSid | 被添加的成员 SID | 确认谁被添加 |
| MemberName | 被添加的成员名 | 确认被添加的账户 |

**应急场景**:
- **跨域权限提升**: 账户被添加到跨域通用组 → 可能是权限提升
- **组策略影响**: 通用组成员变更影响多个域 → 影响范围大
- **关联检测**: 4756 → 4624 (登录) → 添加成员后立即使用新权限
- **与 4728 区别**: 4728 是全局组，4756 是通用组 → 4756 影响范围更大

## 十一、补充系统事件

### 7045 - 服务创建 (详细版)

**级别**: 信息  
**描述**: 新服务被安装到系统中。此事件在 System 日志中生成，与 Security 日志的 4697 (服务安装) 不同，7045 由服务控制管理器生成，包含更详细的配置信息。

**重要字段**:

| 字段 | 说明 | 应急场景 |
|------|------|----------|
| ServiceName | 服务名称 | 确认安装的服务 |
| ImagePath | 服务可执行文件路径 | 分析服务来源 |
| ServiceType | 服务类型 (用户模式服务/内核驱动) | 分析服务类型 |
| StartType | 启动类型 (自动/手动/禁用) | 分析服务行为 |
| AccountName | 服务运行账户 | 确认服务权限 |

**与 4697 的区别**:

| 特征 | 4697 (Security) | 7045 (System) |
|------|-----------------|---------------|
| 日志类型 | 安全日志 | 系统日志 |
| 生成时机 | 服务安装审核 | 服务控制管理器记录 |
| 字段详细度 | 基本 | 更详细 (含 ServiceType) |
| 审核策略依赖 | 需要"审核系统事件" | 无需特殊审核策略 |

**StartType 值**:

| 值 | 类型 | 说明 |
|----|------|------|
| auto | 自动 | 系统启动时自动运行 |
| demand | 手动 | 需要时启动 |
| disabled | 禁用 | 不启动 |

**ServiceType 值**:

| 值 | 类型 | 说明 |
|----|------|------|
| user mode service | 用户模式服务 | 普通服务 |
| kernel mode driver | 内核模式驱动 | 驱动程序 |
| file system driver | 文件系统驱动 | 文件系统驱动 |

**应急场景**:
- **恶意服务**: ImagePath 指向 Temp、AppData 或网络路径 → 高度可疑
- **随机名称**: ServiceName 为随机字符串 → 可能是攻击者创建
- **高权限账户**: AccountName 为 LocalSystem 或 NetworkService → 权限过高
- **自动启动**: StartType=auto + 异常路径 → 可能是持久化机制
- **与 4697 关联**: 4697 和 7045 同时出现 → 确认服务安装
- **SC 工具**: ServiceName 包含 "sc" 或 ImagePath 包含 cmd.exe → 可能是 sc.exe 创建
- **常见恶意服务名**: update, security, service, helper, system

## 五、使用建议

1. **审核策略配置**: 确保启用相关审核策略以生成这些事件
2. **日志收集**: 使用 WinLogAnalyzer-Go 或其他 SIEM 工具收集和分析这些事件
3. **告警规则**: 基于应急场景创建告警规则
4. **白名单管理**: 为合法操作创建白名单以减少误报
5. **关联分析**: 将多个事件关联以检测完整的攻击链路
